# Решение ошибок при запуске

## Ошибки, которые вы видите:

### 1. `Cannot GET /` на backend
**Это нормально!** Backend API не имеет корневого маршрута, все маршруты начинаются с `/api`.

**Решение:** Я добавил корневой маршрут в backend для проверки. Теперь при обращении к `http://localhost:3001/` вы увидите JSON с информацией о сервере.

### 2. CSP ошибка с Chrome DevTools
```
Connecting to 'http://localhost:3001/.well-known/appspecific/com.chrome.devtools.json' violates CSP
```

**Это не критично!** Это просто предупреждение браузера Chrome о попытке подключения DevTools. Это не влияет на работу приложения.

**Решение:** Можно игнорировать. Это стандартное поведение Chrome DevTools.

### 3. `404 (Not Found)` ошибка
**Возможные причины:**
- Браузер пытается загрузить ресурс, которого нет
- Frontend пытается обратиться к несуществующему API endpoint

## Что нужно сделать:

### 1. Перезапустите backend
После добавления корневого маршрута нужно перезапустить backend:

```cmd
# Остановите текущий процесс (Ctrl+C)
# Затем запустите снова:
npm run dev:all
```

### 2. Проверьте, что серверы работают:

**Backend:** Откройте в браузере http://localhost:3001
- Должно показать: `{"status":"ok","message":"Task Manager API is running","version":"1.0.0"}`

**Frontend:** Откройте в браузере http://localhost:3000
- Должно перенаправить на `/auth/login` или `/projects`

### 3. Проверьте консоль браузера
Откройте DevTools (F12) и посмотрите:
- Какие именно ресурсы возвращают 404
- Есть ли ошибки JavaScript

## Если проблемы сохраняются:

1. **Очистите кеш браузера:**
   - Нажмите Ctrl+Shift+Delete
   - Очистите кеш и перезагрузите страницу

2. **Проверьте, что оба сервера запущены:**
   ```cmd
   # Должны быть запущены:
   # - Next.js на порту 3000
   # - Backend на порту 3001
   ```

3. **Проверьте переменные окружения:**
   - `.env.local` должен содержать: `NEXT_PUBLIC_API_URL=http://localhost:3001/api`
   - `backend/.env` должен содержать правильные настройки

## Ожидаемое поведение:

1. Открываете http://localhost:3000
2. Если не авторизованы → перенаправление на `/auth/login`
3. После входа → перенаправление на `/projects`
4. Backend API доступен на http://localhost:3001/api/...

## Проверка работоспособности:

**Тест Backend:**
```bash
curl http://localhost:3001/
# Должен вернуть: {"status":"ok",...}
```

**Тест Frontend:**
- Откройте http://localhost:3000
- Должна загрузиться страница входа или проектов
