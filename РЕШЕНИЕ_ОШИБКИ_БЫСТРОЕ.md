# Быстрое решение ошибки git-remote-https.exe

## Проблема
Ошибка: `git-remote-https.exe - Application Error` - ошибка доступа к памяти.

## Решение: Использовать SSH (обходит проблему)

SSH не использует `git-remote-https.exe`, поэтому полностью решает проблему.

### Шаг 1: Создайте SSH ключ

Откройте **Git Bash** (не PowerShell!) и выполните:

```bash
ssh-keygen -t ed25519 -C "mar-ka4@github"
```

Нажмите **Enter 3 раза** (для всех вопросов).

### Шаг 2: Скопируйте публичный ключ

В Git Bash выполните:

```bash
cat ~/.ssh/id_ed25519.pub
```

Скопируйте весь вывод (начинается с `ssh-ed25519`).

### Шаг 3: Добавьте ключ на GitHub

1. Откройте: https://github.com/settings/keys
2. Нажмите **"New SSH key"**
3. **Title**: введите любое название (например, "My PC")
4. **Key**: вставьте скопированный ключ
5. Нажмите **"Add SSH key"**

### Шаг 4: Измените URL репозитория на SSH

В PowerShell или Git Bash:

```powershell
cd c:\Users\RAZER\Desktop\task-manager
git remote set-url origin git@github.com:mar-ka4/task-manager.git
```

### Шаг 5: Отправьте код

```powershell
git push -u origin main
```

---

## Альтернатива: GitHub Desktop

Если SSH не подходит, используйте GitHub Desktop:

1. Скачайте: https://desktop.github.com/
2. Установите и войдите в GitHub
3. File → Add Local Repository
4. Выберите `c:\Users\RAZER\Desktop\task-manager`
5. Нажмите "Publish repository"

---

## Если ничего не помогает

Переустановите Git:
1. Удалите Git через "Программы и компоненты"
2. Скачайте: https://git-scm.com/download/win
3. Установите заново
4. Перезагрузите компьютер
